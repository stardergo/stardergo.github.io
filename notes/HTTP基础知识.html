简介：超文本传输协议（Hypertext Transfer Protocol，HTTP）是一个简单的请求-响应协议，它通常运行在TCP之上。它指定了客户端可能发送给服务器什么样的消息以及得到什么样的响应。 请求和响应消息的头以ASCII形式给出；而消息内容则具有一个类似MIME的格式。 超文本传输协议是一种用于分布式、协作式和超媒体信息系统的应用层协议，是万维网WWW（World Wide Web）的数据通信的基础。

在计算机网络中，HTTP（HyperText Transfer Protocol）请求数据包是用来在客户端和服务器之间传输数据的一种格式。这些数据包遵循一定的结构，包括请求行、请求头和可选的请求体。
HTTP请求数据包的基本结构

HTTP请求包格式：
1. 请求行（Request Line）
2. 请求头（Headers）
3. 请求体（Body，可选）

其中：

1. 请求行（Request Line）
请求行是HTTP请求的第一行，它包含了三个主要部分：
‌方法（Method）‌：如GET、POST、PUT、DELETE等，指明了要对资源执行的操作类型。
‌请求的目标资源（Request-URI）‌：通常是资源的路径，例如/index.html。
‌HTTP版本（HTTP Version）‌：如HTTP/1.1。
例如：
GET /index.html HTTP/1.1

2. 请求头（Headers）
请求头跟随在请求行之后，以键值对的形式提供额外的信息，如：
Host‌：指定请求的服务器的域名。
‌User-Agent‌：发送请求的客户端信息。
‌Accept‌：客户端可接受的内容类型。
‌Content-Type‌：请求体的媒体类型，例如application/x-www-form-urlencoded或application/json。
‌Content-Length‌：请求体的长度（如果存在）。
例如：
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.3
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8
Content-Type: application/x-www-form-urlencoded
Content-Length: 27

3. 请求体（Body，可选）（注意：在编辑请求包时body部分要与前面空一行）
请求体是可选的，它包含了要发送给服务器的数据。例如，在POST请求中，表单数据或JSON数据会被包含在请求体中。
例如，一个POST请求的请求体可能看起来像这样：
name=John+Doe&age=25
或者，如果是JSON格式：
{
"name": "John Doe",
"age": 25
}

完整的HTTP GET请求示例

GET /index.html HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.3
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8

完整的HTTP POST请求示例（包含请求体）
POST /submit HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.3
Accept: */*
Content-Type: application/x-www-form-urlencoded
Content-Length: 27

name=John+Doe&age=25
